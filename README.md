# sfc — Symfony Companion CLI

Rust CLI for post-build analysis and optimization of Symfony projects running on PHP-FPM.

Operates on artifacts generated by `cache:warmup --env=prod`. Never touches source code.

## Install

```bash
cargo install --path .
```

Or grab a binary from [Releases](https://github.com/ptondereau/sfc/releases).

## Usage

```bash
composer install --no-dev
bin/console cache:warmup --env=prod

sfc analyze .          # detect dead services, unused listeners, dead routes
sfc optimize -O2 .     # strip unreachable factory files from cache
sfc preload .          # generate topologically sorted preload.php
sfc init .             # bootstrap sfc.toml
```

## Commands

### `sfc analyze`

Parses the compiled container and routing cache. Surfaces:

- **Dead services** — registered but never injected
- **Container weight** — memory footprint estimation
- **Unused event listeners** — listening to events never dispatched in `src/`
- **Always-loaded voters** — non-lazy voters adding startup cost
- **Dead routes** — routes pointing to missing controllers

```bash
sfc analyze /app                    # colored terminal output
sfc analyze /app --format json      # JSON for CI pipelines
```

Exit code 1 on critical findings.

### `sfc optimize`

Strips unnecessary factory files from `var/cache/prod/`. Automatic backup before any change.

```bash
sfc optimize /app                   # level 1: remove dead service factories
sfc optimize -O2 /app               # level 2: also remove unreachable factories
sfc optimize -O2 --dry-run /app     # preview without modifying
sfc optimize --restore /app         # rollback from backup
```

On a real production Symfony app (1,977 services), level 2 removes **1,087 factory files** and frees **1.7 MB**.

### `sfc preload`

Generates `preload.php` for OPcache. Scans PHP files, builds an inheritance graph, and topologically sorts classes so parents load before children.

```bash
sfc preload /app                        # scan cache + vendor
sfc preload /app --no-vendor            # cache only
sfc preload /app --output preload.php   # custom output path
```

Point `opcache.preload` at the generated file.

### `sfc init`

Creates `sfc.toml` with auto-detected Symfony version and commented defaults. Does not require a warmed cache.

```bash
sfc init /app
```

## Configuration

Optional `sfc.toml` at the project root:

```toml
[project]
# root = "."
# cache_dir = "var/cache/prod"
# src_dir = "src"

[analyze]
format = "terminal"
exclude_bundles = []

[preload]
# output = "var/cache/prod/preload.php"
# exclude_namespaces = ["App\\Tests"]
# max_classes = 0
# scan_vendor = true
```

All fields are optional. Without `sfc.toml`, everything is auto-detected.

## Integration

Add to your build pipeline after `cache:warmup`:

```bash
sfc analyze --format json . | jq '.summary'
sfc optimize -O2 .
sfc preload .
```
